# Vector Search API Documentation

## Overview

Vector search enables **semantic similarity matching** on resumes using embeddings generated by the Mistral AI model. Unlike keyword-based search (BM25), vector search finds conceptually similar documents even if they don't share exact keywords.

- **Similarity Metric**: Cosine (normalized dot product)
- **Embedding Model**: Mistral AI (mistral-embed)
- **Vector Dimension**: 1024
- **Implementation**: Hybrid (MongoDB Atlas with application-level fallback for robustness)

---

## Endpoints

### 1. Basic Vector Search

**POST** `/v1/search/vector`

Find semantically similar resumes using a text query.

#### Request

```json
{
  "query": "senior python developer with machine learning",
  "limit": 10,
  "skip": 0
}
```

**Parameters:**
- `query` (string, required): Natural language search query describing desired resume characteristics
- `limit` (number, optional): Results per page (1-100, default: 10)
- `skip` (number, optional): Number of results to skip for pagination (default: 0)

#### Response

```json
{
  "status": "success",
  "message": "Vector search found 10 similar results",
  "data": [
    {
      "_id": "507f1f77bcf86cd799439011",
      "name": "John Doe",
      "email": "john@example.com",
      "role": "Senior Python Developer",
      "company": "Tech Corp",
      "skills": "Python,Machine Learning,TensorFlow,PyTorch,AWS",
      "score": 0.8234
    },
    {
      "_id": "507f1f77bcf86cd799439012",
      "name": "Jane Smith",
      "email": "jane@example.com",
      "role": "ML Engineer",
      "company": "AI Inc",
      "skills": "Python,Deep Learning,NLP,Scikit-learn",
      "score": 0.7956
    }
  ],
  "metadata": {
    "total": 10,
    "limit": 10,
    "skip": 0,
    "metric": "cosine"
  }
}
```

**Score Interpretation:**
- `1.0`: Identical match
- `0.9+`: Highly similar (excellent match)
- `0.7-0.9`: Similar (good match)
- `0.5-0.7`: Somewhat similar (acceptable match)
- `<0.5`: Low similarity

### 2. Advanced Vector Search with Threshold

**POST** `/v1/search/vector/advanced`

Find only resumes with similarity above a specified threshold.

#### Request

```json
{
  "query": "machine learning engineer with 5+ years experience",
  "scoreThreshold": 0.75,
  "limit": 10,
  "skip": 0
}
```

**Parameters:**
- `query` (string, required): Natural language search query
- `scoreThreshold` (number, optional): Minimum similarity score (0-1, default: 0.5)
- `limit` (number, optional): Results per page (1-100, default: 10)
- `skip` (number, optional): Number of results to skip (default: 0)

#### Response

```json
{
  "status": "success",
  "message": "Advanced vector search found 7 results with score >= 0.75",
  "data": [
    {
      "_id": "507f1f77bcf86cd799439011",
      "name": "John Doe",
      "email": "john@example.com",
      "role": "Senior ML Engineer",
      "company": "Tech Corp",
      "skills": "Python,TensorFlow,Machine Learning",
      "score": 0.8567
    }
  ],
  "metadata": {
    "total": 7,
    "limit": 10,
    "skip": 0,
    "scoreThreshold": 0.75,
    "metric": "cosine"
  }
}
```

### 3. Vector Search Statistics

**GET** `/v1/search/vector/stats`

Get statistics about the vector search setup and indexed documents.

#### Request

```
No request body required
```

#### Response

```json
{
  "status": "success",
  "message": "Vector search statistics retrieved",
  "data": {
    "totalDocuments": 92,
    "documentsWithEmbeddings": 90,
    "embeddingDimension": 1024,
    "similarityMetric": "cosine",
    "searchMethod": "hybrid (MongoDB Atlas with application-level fallback)"
  }
}
```

**Fields:**
- `totalDocuments`: Total resumes in the database
- `documentsWithEmbeddings`: Resumes with generated embeddings (ready for search)
- `embeddingDimension`: Vector size (1024 for Mistral embeddings)
- `similarityMetric`: Distance metric used for comparison
- `searchMethod`: Implementation method (hybrid for robustness)

### Example 3: Pagination

```bash
# First page
curl -X POST http://localhost:5000/v1/search/vector \
  -H "Content-Type: application/json" \
  -d '{"query": "python developer", "limit": 10, "skip": 0}'

# Next page
curl -X POST http://localhost:5000/v1/search/vector \
  -H "Content-Type: application/json" \
  -d '{"query": "python developer", "limit": 10, "skip": 10}'
```

### Example 4: JavaScript Integration

```javascript
async function searchSimilarCandidates(query) {
  const response = await fetch('http://localhost:5000/v1/search/vector', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      query: query,
      limit: 10,
      skip: 0
    })
  });
  
  const result = await response.json();
  
  if (result.status === 'success') {
    result.data.forEach(candidate => {
      console.log(`${candidate.name} - Score: ${(candidate.score * 100).toFixed(1)}%`);
    });
  }
}

searchSimilarCandidates('senior backend engineer with microservices');
```

## Search Query Tips

### Good Queries

✅ **Descriptive**: "senior backend engineer with 5+ years in microservices and AWS"  
✅ **Skill-focused**: "Python, Django, REST APIs, PostgreSQL"  
✅ **Role-based**: "Data scientist specializing in machine learning and big data"  
✅ **Experience-based**: "Project manager with Agile and Scrum experience"  

### Queries to Avoid

❌ **Too vague**: "engineer"  
❌ **Too specific**: "worked at company X from 2020-2022"  
❌ **Multiple unrelated skills**: "java and photography and cooking"

## Combining with BM25 Search

For hybrid search combining semantic + keyword matching:

```javascript
async function hybridSearch(query) {
  // Get semantic matches
  const vectorResults = await fetch('http://localhost:5000/v1/search/vector', {
    method: 'POST',
    body: JSON.stringify({ query, limit: 20 })
  }).then(r => r.json());
  
  // Get keyword matches
  const bm25Results = await fetch('http://localhost:5000/v1/search/bm25', {
    method: 'POST',
    body: JSON.stringify({ query, limit: 20 })
  }).then(r => r.json());
  
  // Merge and re-rank results
  const merged = new Map();
  
  // Add vector search results (weight: 0.6)
  vectorResults.data.forEach((r, i) => {
    merged.set(r._id, { ...r, score: r.score * 0.6 });
  });
  
  // Add BM25 results (weight: 0.4)
  bm25Results.data.results.forEach((r, i) => {
    const id = r._id;
    const existing = merged.get(id);
    if (existing) {
      existing.score += (r.score / 100) * 0.4; // Normalize BM25 score
    } else {
      merged.set(id, { ...r, score: (r.score / 100) * 0.4 });
    }
  });
  
  // Sort by combined score
  return Array.from(merged.values())
    .sort((a, b) => b.score - a.score)
    .slice(0, 10);
}
```

## Implementation Details

### Architecture

```
Query String
    ↓
Embedding Generation (Mistral API)
    ↓
MongoDB Vector Search attempt
    ↓ (if unavailable or no results)
Application-Level Similarity (Fallback)
    ↓
Cosine Similarity Calculation
    ↓
Sorted Results (highest score first)
```

### Supported Search Methods

1. **MongoDB Atlas Vector Search** (primary)
   - Approximate nearest neighbor search
   - Optimized for large datasets
   - Fallback available if index not ready

2. **Application-Level Similarity** (fallback)
   - Exact nearest neighbor using cosine similarity
   - 100% recall guarantee
   - Used when MongoDB vector search unavailable

## Rate Limiting

- **Vector Search Queries**: No limit (uses cached results)
- **Recommended batch size**: 1-20 queries per request
